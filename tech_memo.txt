🔶TypeScriptメモ

■「: ◯◯」の書き方の違い。
・なぜ下記: Promise<Blog[]>は「戻り値」で、
  async function getBlogPosts(): Promise<Blog[]> {

  }
・なぜ下記posts: Blog[]は「型指定」になるのか？
  const posts: Blog[] = await getBlogPosts();

⬇︎
・同じく「: ◯◯」で似てる…。
・違いは何❓

⬇︎
🍏解説。
関数functionの後ろの : 型  ➡️ 「戻り値」の型
変数const の後ろの : 型  ➡️ 「変数」の型
になります。

🔶静的パスを生成する

🍏「静的パス」とは？
・「動的ルート」を持つページを『ビルド時に静的に生成して、あらかじめHTMLを生成する。その結果"ユーザーがアクセスした時にすぐに表示してくれる。"』
generateStaticParamsを使う場合(SSG)になる。


🍏generateStaticParamsを使うべき時(SSG)は？
・URLが事前に決まっているページ。
・内容の更新が頻繁ではないページ。
①ブログ記事サイト
②商品ページ(ECサイト)
③コーポレートサイト(/aboutページなど)


🍏generateStaticParamsを使わないサイトは？(SSR)
・リクエストのたびに内容が変わる
。常に最新のデータを確実に表示させるべきサイト
①SNS
②ダッシュボード
③ショッピングカート

🔶getCategoryPostsについて。


・「特定カテゴリの記事一覧」を取得するために『getCategoryPosts()』を使用します。

⬇️

🍏例として「URLを叩くと・・・」
①URL:　category/react　を開く。
      ↓
②params = "例:react"になる。
      ↓
③「②」をgetCategoryPosts関数に渡します。
const { slug } = await params;
const posts = await getCategoryPosts(slug);

・category/[slug].page.tsxの
『await getCategoryPosts(slug)』の(slug)が、

・libs/blog.tsの
export async function getCategoryPosts(categoryId: string): Promise<Blog[]>
の、(categoryId: string) = "例:react"として受け取ります。

      ↓
⑤その後「microCMS」に送信する。
filters: `category[contains]${categoryId}`　でフィルター処理。
categoryId = 例:react
そして
category[contains]例:react
になり、
      ↓
✅reactカテゴリの記事だけ取得して『記事一覧として表示します。』
